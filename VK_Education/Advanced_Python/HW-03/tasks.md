# ЗАДАНИЕ

Это домашнее задание предназначено для глубокого погружения во внутренние механизмы Python, которые служат фундаментом для таких популярных технологий, как Django ORM, SQLAlchemy, Pydantic и веб-фреймворки уровня FastAPI. 

Понимание принципов работы дескрипторов и метаклассов позволяет разработчику перестать воспринимать декларативный синтаксис этих библиотек как “магию” и начать осознанно проектировать сложные архитектуры, системы валидации и автоматической регистрации компонентов.

В рамках этого домашнего задания необходимо выполнить 4 шага:
1. Реализовать базовый дескриптор типа.
2. Создать дескриптор с валидацией значений.
3. Создать метакласс-реестр.
4. Собрать все созданные компоненты воедино.

---

## Требования к коду

* Необходимо использовать стандартную библиотеку.
* Код должен соответствовать PEP 8.
* Запрещено возвращать `None` при ошибке, необходимо выбрасывать указанные исключения.
* Допускается отсутствие строгих тайп хинтов, но их наличие приветствуется.

## Домашнее задание №3

### Шаг 1. Базовый дескриптор типа
**Задача:** Реализовать класс-дескриптор `TypedProperty`, который при присваивании значения атрибуту проверяет его тип.

1. Дескриптор должен принимать ожидаемый тип в `__init__`.
2. Должны быть реализованы методы `__get__`, `__set__`, `__set_name__`.
3. Хранение значения должно происходить в экземпляре владельца (через `instance.__dict__`), а не внутри дескриптора.

**Важно:** Если тип не совпадает с ожидаемым — необходимо выбрасывать `TypeError`.

### Шаг 2. Дескриптор с валидацией значений
**Задача:** Создать класс `ValidatedProperty`, унаследованный от `TypedProperty`, с поддержкой ограничений значений.

1. Добавьте возможность передачи ограничений (например, `min`, `max` для чисел или `min_length` для строк).
2. Требуется наличие поддержки как минимум одного типа валидации (числовой диапазон ИЛИ длина строки).

**Важно:** При нарушении ограничения необходимо выбрасывать `ValueError`.

### Шаг 3. Метакласс-реестр
**Задача:** Создать метакласс `RegistryMeta`, который автоматически регистрирует классы в глобальном словаре.

1. Метакласс должен наследоваться от `type`.
2. Реестр должен быть доступен через атрибут метакласса (например, `RegistryMeta.registry`).
3. Необходимо исключить дублирование имен в реестре или обрабатывать это явно.

### Шаг 4. Интеграция
**Задача:** Собрать все компоненты в базовый класс `Model` с автоматическим сбором информации о полях.

1. Создайте расширенный метакласс `ModelMeta`, наследующий `RegistryMeta`.
2. Метакласс должен сканировать атрибуты класса при наследовании от `Model`.
3. Все найденные дескрипторы должны сохраняться в словарь `_fields` (имя атрибута -> объект дескриптора).
4. Создайте базовый класс `Model`, использующий `ModelMeta`.
