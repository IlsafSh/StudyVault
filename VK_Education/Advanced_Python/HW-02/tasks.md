# ЗАДАНИЕ

Объектная модель Python. Стек

В реальной разработке часто требуется создание собственных структур данных, когда встроенные типы (list, deque) не обеспечивают необходимый уровень контроля или семантики.

Представь, что ты разрабатываешь веб-фреймворк. Для управления локальным состоянием сессий базы данных используется структура данных стек.

Твоя задача - реализовать упрощенную версию такого менеджера состояний.

Ключевая особенность: состояние должно накапливаться в процессе работы, но гарантированно очищаться при выходе из определенной области видимости (контекста).

---

## Требования к коду

* Необходимо использовать стандартную библиотеку.
* Код должен соответствовать PEP 8.
* Запрещено возвращать `None` при ошибке, необходимо выбрасывать пользовательские исключения.

---

## Домашнее задание №2

### Шаг 1. Базовая логика и обработка ошибок
**Задача:** Реализовать фундаментальные методы жизненного цикла стека.

1. Метод `push(value)`: добавляет элемент.
2. Метод `pop()`: удаляет и возвращает верхний элемент.
3. Класс исключения `StackIsEmpty`: должен наследоваться от `Exception`.

**Важно:** `pop()` должен выбрасывать `StackIsEmpty`, если стек пуст, а не стандартный `IndexError`.

### Шаг 2. Визуализация и размер (магические методы)
**Задача:** Сделать объект удобным для отладки и мониторинга (как логи в production-среде).

1. Метод `__len__()`: возвращает количество элементов (для функции `len()`).
2. Метод `__str__()`: возвращает строку вида `Stack(1, 2, 3)`.
3. Метод `__repr__()`: возвращает строку, которую можно использовать для воссоздания объекта (например, `Stack([1, 2, 3])`).

### Шаг 3. Протокол коллекции
**Задача:** Реализовать поведение, позволяющее использовать стек в циклах и условиях (инспекция состояния).

1. Метод `__iter__()`: возвращает итератор (позволяет использовать `for x in stack`).
2. Метод `__contains__(value)`: позволяет использовать оператор `in`.
3. Метод `__getitem__(index)`: доступ по индексу (например, `stack[0]`).

**Важно:** `__getitem__` должен поддерживать отрицательную индексацию и выбрасывать `IndexError` стандартным образом.

### Шаг 4. Расширенные протоколы
**Задача:** Превратить класс в полноценный инструмент для управления временным состоянием (Scoped State).

1. **Контекстный менеджер.** Реализовать `__enter__` и `__exit__`.
    * `__enter__` возвращает сам объект стека.
    * `__exit__` полностью очищает стек при выходе из блока `with`.
    * `__exit__` должен корректно обрабатывать исключения (не скрывать их, но гарантировать очистку).

2. **Сравнение.** Реализовать `__eq__`.
    * Два стека равны, если их содержимое идентично (порядок и значения совпадают).
